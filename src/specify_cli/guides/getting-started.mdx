# Getting Started with SpecifyX

Welcome to SpecifyX! This guide will help you get up and running with specification-driven development.

## Installation

### Using uv (Recommended)

```bash
uv tool install specifyx
```

### Using pipx

```bash
pipx install specifyx
```

### Using pip

```bash
pip install specifyx
```

## Quick Start

### 1. Create Your First Project

```bash
specifyx init my-awesome-project
```

You'll be prompted to:
- Choose an AI assistant (Claude, Gemini, or Copilot)
- Select a branch naming pattern
- Confirm project creation

### 2. Navigate to Your Project

```bash
cd my-awesome-project
```

### 3. Create Your First Feature

Use the generated Python script:

```bash
python .specify/scripts/create-feature.py "Add user authentication"
```

This creates:
- A new feature branch
- A specification file in `specs/`
- A git branch following your naming pattern

### 4. Write Your Specification

Edit the generated spec file to define:
- User stories and scenarios
- Functional requirements
- Success criteria

### 5. Plan Implementation

```bash
python .specify/scripts/setup-plan.py setup
```

This generates:
- Implementation plan
- Technical design
- Task breakdown

### 6. Generate Tasks

```bash
python .specify/scripts/generate-tasks.py
```

Creates actionable tasks for implementation.

## Core Concepts

### Specification-Driven Development

SpecifyX follows a three-phase lifecycle:

1. **Specify**: Define what to build
2. **Plan**: Design how to build it
3. **Tasks**: Break down into executable tasks

### AI Assistant Integration

SpecifyX generates AI-specific commands for:
- Claude (`/specify`, `/plan`, `/tasks`)
- Gemini (custom prompts)
- Copilot (workspace commands)

### Branch Naming Patterns

Choose from patterns like:
- `feature/{feature-name}`
- `001-feature-name`
- `hotfix/{bug-id}`
- Custom patterns

## Tips

1. **Use Interactive Mode**: Run commands without options for guided prompts
2. **Check System**: Run `specifyx check` after installation
3. **Customize Templates**: Edit `.specify/templates/` for your workflow
4. **Version Control**: Commit `.specify/config.toml` to share settings
